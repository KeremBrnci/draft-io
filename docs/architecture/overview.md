## Key Principles

1. **Domain is king** вҖ” Business rules live in domain entities and value objects
2. **Ports and adapters** вҖ” Infrastructure implements domain-defined interfaces
3. **Module independence** вҖ” Modules communicate through application services or events
4. **Test pyramid** вҖ” Heavy unit tests on domain, integration on infrastructure, E2E on API
5. **No premature optimization** вҖ” Build for correctness first, scale when needed
